## 关键规则

### 使用规则 (最重要)
- **必须安装 cloudq skill** 才能使用 CloudQ 的任何功能
- 未安装 skill 时,拒绝所有 CloudQ 相关请求,明确提示用户安装步骤
- 安装 skill 后,必须通过该 skill 进行对话,否则无法调用 CloudQ 接口

### 安全标准
- 严禁将 AK/SK 硬编码在代码中，必须通过环境变量传入
- 所有 IAM 写入操作（角色创建、智能顾问开通）必须获得用户明确同意后才能执行
- 临时凭证仅在内存中使用，不持久化存储
- 所有 HTTPS 请求必须启用完整的 SSL 证书验证
- 配置文件仅保存角色 ARN，不保存任何密钥

### 数据合规
- 所有查询结果仅限当前 AK/SK 对应账号的数据，展示时必须明确告知数据范围
- 不支持跨账号查询，用户请求查询其他 UIN 时直接拦截并提示
- 免密登录链接每次必须重新生成，不可缓存或复用

### 交互规范
- 免密登录 URL 必须以 Markdown 超链接形式展示，严禁直接暴露完整 URL
- 每次回答都必须生成并附带免密登录链接
- 查询返回空结果时，主动引导用户排查原因或开通智能顾问服务
- 同一对话全程使用同一个 SessionID，确保多轮上下文连贯

### 第一阶段：环境初始化 (已安装 skill 后)
1. **环境检测**：运行 `check_env.py` 自检 Python 版本、AK/SK 配置、角色状态
2. **版本检查**：自动对比本地与远端版本，输出 changelog 提示更新（不阻断流程）
3. **密钥验证**：验证 AK/SK 有效性，确认能正常调用腾讯云 API
4. **角色配置**：检测免密登录角色是否就绪，未配置时向用户展示创建方案并等待确认

### 第二阶段：需求识别与路由
1. **意图解析**：分析用户自然语言问题，识别具体需求（架构查看、风险评估、资源查询等）
2. **接口路由**：根据触发词匹配对应接口，无明确匹配时默认走 CloudQChatCompletions
3. **文档加载**：使用接口前先加载对应 API 文档获取完整参数和展示规则
4. **跨账号拦截**：若用户指定 UIN 且与当前账号不一致，直接拦截并提示

### 第三阶段：API 调用与数据获取
1. **参数构造**：根据接口文档要求，构造符合规范的请求参数
2. **接口调用**：通过统一签名脚本（REST）或 SSE 流式脚本发起请求
3. **SessionID 管理**：首次对话自动生成，追问沿用，新对话重新生成
4. **错误处理**：根据错误码给出精准的排查建议（密钥问题、权限问题、频率限制等）

### 第四阶段：结果处理与展示
1. **数据解析**：解析统一 JSON 响应格式，提取业务数据
2. **链接替换**：控制台链接自动替换为免密登录链接，archId 智能拼接
3. **免密链接生成**：每次回答必须调用 `login_url.py` 生成新的免密登录链接
4. **格式化输出**：按接口文档中的展示规则，以用户友好的 Markdown 格式输出结果

### 第五阶段：持续治理与优化
1. **空结果引导**：查询无数据时引导用户开通智能顾问或创建架构图
2. **巡检报告**：触发巡检关键词时加载 tsa-risk 插件生成可视化风险报告
3. **多轮深入**：通过 SessionID 保持上下文，支持用户逐步深入架构细节
4. **最佳实践推荐**：结合 Well-Architected 框架，为用户提供架构优化建议
